## Problem

A single curriculum author needs to efficiently develop, edit, and publish a Science of Reading-aligned phonics curriculum for multilingual English learners — particularly middle-school newcomers. Current process is manual and slow; creating a single unit takes hours. The resulting materials need to be print-ready and publication-quality.

The curriculum differs from traditional native-speaker phonics programs because it intentionally accounts for language-transfer opportunities (especially from Spanish), the developmental needs of older learners, and the requirement for meaningful, age-appropriate vocabulary.

## Target User

**Primary:** Curriculum author (sole author) — creates, edits, and publishes all content

**Secondary:** Teachers, reading specialists, English learner instructors — consume the published curriculum

**End Learners:** Middle-school newcomer English learners, intermediate ELs with phonics gaps, long-term ELs missing foundational phonics skills

## Goals

- Reduce unit creation time from hours to minutes
- Generate consistent, print-ready student workbook units, teacher manual content, assessments, and companion website resources
- Allow the author to directly edit all generated content before publication
- Produce curriculum suitable for commercial publication
- The software is a production tool — not the product itself

## Core Features

1. Curriculum generation (workbook pages, teacher guidance, vocabulary, decodable passages, assessments)
2. Vocabulary management with required image per word
3. Photo/asset management (upload, replace, reuse, approve)
4. Articulation support
5. Orthographic mapping automation
6. Language transfer support (Spanish-aware, not Spanish-exclusive)
7. Assessment generation (beginner and intermediate levels)
8. Direct editing of all generated content
9. PDF export (student workbook + teacher manual)
10. Template-based consistent layouts

## Constraints

- Single-author tool — no collaboration features needed
- Not a commercial software product — internal production tool only
- Vocabulary must be age-appropriate for middle school
- Every vocabulary word must have an associated image
- AI output must be structured data (curriculum objects) — templates render final pages, not AI directly
- Google Slides export is a future feature, not MVP scope

## Open Questions

- **Image sourcing** — spec requires an image per vocabulary word; sourcing method undefined. Options: manual upload, AI-generated (DALL-E), or stock photo API (Unsplash). Architectural decision needed before Phase 4.
- **PDF rendering library** — not specified in docs. Evaluate `@react-pdf/renderer` (React-native, no headless browser) vs Puppeteer/Playwright (full HTML rendering, heavier). Decision needed before Phase 5.
- **Auth scope** — single-user system with Supabase Auth implies login flow. Clarify: single hard-coded account, invite-only, or email/password?
- **Google Slides export** — listed as "Future" with no timeline; deferring past Phase 5 entirely.
